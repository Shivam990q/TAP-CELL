## Hiring process — Front End Developer (from public interview experiences, GeeksforGeeks)
> Typically **8–9 elimination rounds** (each is a cut): ~4 online (home/college) + ~5 at the
> Gurugram office. Assessments run from campus labs with **webcam + mic on**.

Representative flow (varies slightly by year):
1. **MCQ round** — ~50–70 questions, 1 hour, **negative marking**. Mix: ~10 on C/C++, ~40 on
   HTML/CSS/JavaScript (syntax, output-guessing, pseudo-code). Accuracy > speed.
2. **JS-focused round** — strong emphasis on JavaScript (promises, async), plus HTML/CSS.
3. **Take-home technical assessment** — build a **static web page in pure HTML/CSS/JS**
   (~1 week given).
4. **Timed machine-coding assignment** — build in **< 2 hours, screen-shared**, interviewer
   checks progress every ~20 min.
5. **Technical personal interviews** — HTML, CSS (grilled hard), JavaScript, Computer Networks;
   dry-running code, reasoning.
6. **HR / managerial round** — culture fit, communication, ownership.

## What they screen for (map to Shivam's resume)
- **CSS depth** (flexbox, grid, positioning, specificity, responsive) — most-tested area.
- **Core JavaScript** — closures, promises, async/await, event loop, DOM.
- **HTML5 semantics**, responsive/Bootstrap/Material, SASS/LESS.
- **React fundamentals** (they use React JS & Native) — hooks, state, component design.
- Clean, defect-free code; unit tests; front-end performance.
- Strong communication (they explicitly value blogs/presentations, on/off-shore comms).

## Prep plan for Shivam (2 weeks-ish)
- **CSS drills**: flexbox/grid layouts, centering, specificity, positioning, responsive media queries.
- **JS core**: promises/async-await, closures, `this`, event delegation, debounce/throttle,
  array methods; output-prediction practice (for MCQ + negative marking).
- **Machine coding**: rebuild small UIs from scratch fast (accordion, tabs, carousel, star-rating,
  todo, modal) in vanilla JS + one in React — under time, screen-shared style.
- **Take-home**: a polished static responsive page (pixel-perfect, clean HTML/CSS).
- **CS fundamentals**: basic Computer Networks (HTTP, DNS, TCP/UDP), a little C/C++ output MCQs.
- **Behavioral**: STAR stories (IDC leadership, Hacksagon, client project NETWORKED), why JTG.
