# Career OS — Capability Roadmap

Ideas to maximize benefit. Status: [ ] idea  [~] partial  [x] done.
Pick what you want; I build it.

## A. Make the system truly "know everything" (data depth)
- [ ] **Auto-enrich projects from GitHub READMEs** — read flagship repos, auto-fill each
      `03-projects/*.md` (problem, stack, features, resume bullets). Clears the ~38 pending items.
- [ ] **Skill evidence matrix** — map each skill -> which repo/project proves it (great for interviews).
- [ ] **Tech-stack stats from GitHub** — aggregate languages/topics across all 97 repos to
      validate and rank the skills section by real usage.
- [ ] **STAR behavioral stories** — turn experiences into HR-round stories (challenge-action-result).

## B. Application engine (per job)
- [ ] **JD match & gap analyzer** — paste a JD, get a match score, missing keywords, and which
      projects/skills to feature. ATS keyword-gap report.
- [ ] **Auto-tailor resume** — from a JD, auto-pick relevant projects/skills and generate the
      tailored `.tex` (then you fine-tune).
- [ ] **Cover letter generator** — per-company cover letter from profile + JD.
- [ ] **Deadline tracker / reminders** — list upcoming registration & test deadlines from
      all applications, sorted; warn on near deadlines.

## C. Interview prep (per company)
- [ ] **Auto prep sheet** — per company: rounds, topics, focus areas + a study checklist
      (e.g., JOSH = CSS-heavy, 70-MCQ negative marking).
- [ ] **Question bank** — role-wise (Frontend: CSS/JS/React; SDE: DSA topics) with your weak spots.

## D. Multiple positioning (you target several role types)
- [ ] **Role-specific base resumes** — separate single-page templates for: Frontend,
      Full-Stack, AI/Agentic, SDE. Pick the right base per drive.
- [ ] **Headline/summary variants** — tuned per role family.

## E. Tracking & safety
- [ ] **Application dashboard** — auto status board (applied/OA/interview/offer) from tracker.
- [ ] **Git version control** — init a private git repo so every change is versioned/undoable
      (true "never messed up"); optional push to private GitHub.
- [ ] **Weekly digest** — what's pending, deadlines this week, profile completeness score.

## F. Outputs
- [ ] **Auto-export PDFs** with consistent naming (`Shivam_Gupta_<Role>_<Company>.pdf`).
- [ ] **Portfolio data export** — JSON your portfolio site could consume.

---
### Recommended first (highest leverage)
1. Auto-enrich projects from GitHub READMEs (A) — fills real data, powers every resume.
2. JD match & gap analyzer (B) — instantly tailor smarter per drive.
3. Role-specific base resumes (D) — Frontend / Full-Stack / AI.
4. Git version control (E) — safety net for everything.
