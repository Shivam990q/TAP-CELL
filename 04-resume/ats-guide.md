# ATS Optimization Guide

Quick rules I follow when tailoring a resume to a job description.

## Keyword matching
- Mirror exact terms from the JD (skills, tools, titles). If JD says "React.js", use that.
- Include both spelled-out and acronym forms once: "Natural Language Processing (NLP)".
- Put the most relevant skills near the top.

## Formatting (ATS-safe)
- Single column, standard section headings (Experience, Education, Skills, Projects).
- No tables, text boxes, images, headers/footers for key info.
- Standard fonts; bullet points with "•" or "-".
- Save and submit as PDF unless the portal asks for .docx.

## Bullet formula
`Action verb + what you did + tech/how + quantified impact`
e.g., "Built a REST API in Node.js handling 5k+ daily requests, cutting latency 40%."

## Tailoring checklist (per application)
- [ ] Match title/keywords from JD
- [ ] Reorder skills to match JD priority
- [ ] Pick 3-5 most relevant projects/experiences
- [ ] Quantify everything possible
- [ ] Keep to 1 page (student/new grad)
